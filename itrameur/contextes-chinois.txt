这四种邻居，最不好相处，敬而远之

   2023-04-10 08:51 来源: 小黑的文化茶舍
   发布于：山西省

   原标题：这四种邻居，最不好相处，敬而远之

   这四种邻居，最不好相处，敬而远之

   [355a086902bd446e879c0a15e2a7937b.jpeg]
--
   各家有各家的日子，互不打扰，互不影响，是一份风度。

   遇到好邻居是幸事，遇到坏邻居是灾难。

   邻里之间，低头不见抬头见，和睦相处，彼此都幸福。

   但有的邻居，实在不好相处，只有一个选择——敬而远之。

   01

   “势利眼”的邻居

   有的邻居仗着自己家里更有钱，就很傲慢无礼，这种人很好“辨识”——你对他非常客气，他对你爱搭不理，张口闭口自卖自夸，显得洋洋得意，高人一等。

   这种邻居，骨子里有着很强烈的 攀比心理，从一认识开始，他们就会站在“谁穷谁富”的角度打量你。

   看你富有，他们就亲切热情主动，若是看你家境“一般”，他们就会充满鄙视。
--
   若不想过多联系，大可以不必联系，关起门来过好自己的日子，井水不犯河水。

   还有一种邻居，属于仗势欺人的人，仗着自己有一些财势，专横跋扈，总想把邻居家的一块地、一块砖占为己有。

   听一位小区的住户讲，他家新买的房子，房前的院子与邻居隔着一堵墙，邻居比他家早一步装修，在未经他家允许的情况下，就私占了他家十公分的院子。

   展开全文
--
   02

   “太算计的邻居”

   太算计的邻居，也可以称之为太爱占小便宜的邻居，看你家有什么东西，都想拿来“分享”。

   这样的邻居也有一个很明显的特点，表面虚情假意，说着一些讨好人的话语，实则有自己的小心思，想方设法得到一点好处，得到一点好处便沾沾自喜。

   还有一点就是，太算计的人也太吝啬，得了便宜没问题，一旦吃一点亏就会露出本来面目。

   曾听过这样一件事，有两户人家，彼此是邻居，其中一家总去另一家“蹭饭”，后者觉得是邻里，便不好意思拒绝，经常好酒好肉款待。

   可是反过来，有一次后者去前者家串门，一进门就闻到了肉香，原来是他家正在炖肉，可是前者忽闻后者来，赶紧把肉藏起来，后者一下就懂了，从此再没请前者
   吃过一顿饭。

   邻居之间讲究“礼尚往来”，你送我几根小葱，我送你几个苹果，谁亏了谁赚了，谁心里都明白。

   但有的人，总觉得自己最“精明”，总把别人当傻瓜，其实谁都不傻，只有傻人才做傻事。
--
   邻里之间，彼此心里都明白，谁家的钱都不是大风刮来的，人心都有一杆秤，人情冷暖，各有自知。

   面对一个“太算计”的邻居，只能远离，别无他法。

   03

   “嚼舌根”的邻居

   似乎每一个地方，都有一种人——爱嚼舌根子。
--
   灾乐祸。

   有的邻居就是如此，你也不知道什么地方惹着他了，你过得好，他就很生气，你家里出一点事情，他就放鞭炮。

   你家里出一点事情，他就马上四处宣扬，芝麻大的小事也被他渲染成天大的事，就好像，你已经一败涂地。
--
   你得了感冒，他就敢说你得了癌症，就还像，你已经奄奄一息。

   邻居之间，到底有多大的仇恨，才至于如此愤恨。

   如果有一个“爱嚼舌根”的邻居，说话可要小声点，谨防“隔墙有耳”，面对这样的人，沉默是金，离远一点，一生平安!

   [031c70691fac4be5a948a9fe2207b704.jpeg]
--
   04

   “太计较”的邻居

   有好多楼上楼下的邻居，因为打扰到对方休息产生争执，有的愈演愈烈，吵到不可开交的地步，这种情况早已经屡见不鲜。

   还有的邻居，因为房子装修声音嘈杂，引起其他住户的不满。

   各种各样的问题，导致许多邻里之间发生矛盾，就像夫妻似的，每天同床共枕，也有头碰头的时候。

   既然是邻居，就没有绝对的“不打扰”。

   家家有本难念的经，比如有的家里有孩子，即便家长管教，孩子也有抑制不住天性的时候，在“楼上”蹦跳几下，实属正常。

   邻居的相处之道，关键在于互相尊重，多多礼让，及时沟通。

   有智慧的人，自家孩子闹了，打扰到了邻居，买一兜水果，及时到邻居家赔个不是，大多数人都会理解。

   除非太计较的邻居……

   太计较的邻居往往有一种性格“特色”，就是别人影响他一点都不行，反过来他却可以肆无忌惮的影响别人。

   太计较的邻居，习惯把所有的问题都归咎于别人，却丝毫不在乎别人的感受。

   有时候，邻里之间不能太苛刻，一旦针锋相对，隔着墙打个喷嚏都是错。
--
   人活着，哪有绝对的自由，只有互相给予的空间，不怕距离“太近”，就怕人心太“窄”。

   能成为邻居，是一场缘分，互相友好，互相善待，互相包容，互相帮助。

   一旦碰到太计较的邻居，也只能严于律己，不讲情只讲理，若是连道理也讲不通，就只能互相敌视，形同陌路。

   常言道，茫茫人海，形形色色，遇到好邻居，会少很多困扰。

   一旦遇到糟糕的邻居，倒也不必惊慌失措，过好自己的生活，守好自己的原则，遵守社会的规章，遵从公众的道德，与其保持适当的距离，永不逾越，守好自己的

简介

   本文档介绍开放最短路径优先(OSPF)邻居如何成为完全邻接的常见问题。

先决条件
--
背景信息

   在成功形成OSPF邻接关系时，OSPF邻居会达到FULL邻居状态。要检查邻接关系，请输入 debug ip ospf adj 命令。在输入
   debug 命令之前，请参阅有关 Debug 命令的重要信息。

邻接关系

   路由器是邻居这一事实不足以保证链路状态更新的交换；它们必须形成邻接关系以交换链路状态更新。邻接关系是一种高级邻居关系形式，由路由器形成，这些路
   由器在协商交换参数后交换路由信息。同步链路状态数据库上的视图后，路由器之间将达成 FULL 状态的邻接关系。

   接口类型在邻接关系的形成过程中起着主要作用。例如，点对点链路上的邻居始终会尝试建立邻接关系，而连接到广播介质（如以太网）的路由器只能与接口上的
   一小部分邻居路由器建立邻接关系。

   一旦路由器与邻居形成邻接关系，它首先交换其链路状态数据库的完整副本。邻居也是一样，会轮流交换整个链路状态数据库。在经过几个邻居状态后，路由器会
   完全邻接。

   使用show ip ospf neighbor命令确定OSPF邻居的状态。此命令的输出显示以下内容之一：

     * state = down

   也有其他的 OSPF 状态，但这里显示的状态是 show ip ospf neighbor 命令输出中最常见的几种。有关详细信息以及所有
   OSPF 邻居状态的说明，请参阅 OSPF 邻居状态。

处于 down 状态的邻居

   如果通过OSPF进程删除了邻居，则通过接收HELLO数据包动态发现的邻居可以回退到down状态。例如，当OSPF在比Dead计时器间隔长的一段
   时间内，没有收到来自邻居的HELLO数据包时，此类邻居的down状态是暂时的；它们会进入更高状态或从已知邻居的表中删除。这被称为“被遗忘”。

   通常，在down状态中看到的邻居使用 neighbor 命令。手动配置的邻居始终会出现在 OSPF
   邻居表中。如果OSPF从未收到来自手动配置的邻居的HELLO数据包，或者如果在上一个Dead计时器间隔内没有收到来自邻居的HELLO数据包，则
   手动配置的邻居将列为down。

   注: neighbor 命令只能针对以下网络类型的直连邻居进行配置：
   — 非广播多路访问(NBMA)网络 — 配置有 ip ospf network non-broadcast 命令。
   — 非广播点对多点网络 — 配置有 ip ospf network point-to-multipoint non-broadcast 命令。

   如果邻居处于down状态，请验证该邻居路由器是否处于打开状态、是否处于活动状态以及是否在此接口上正确配置了OSPF。使用 ping
   和 traceroute 命令。使用OSPF邻居路由表 show ip ospf neighbor 命令，并执行与本文档前面的No State
   Revealed部分中列出的配置验证操作相同的操作。

处于 init 状态邻居

   init状态表示路由器收到来自邻居的HELLO数据包，但是尚未建立双向通信。Cisco 路由器会在其 HELLO 数据包的 Neighbor
   字段中显示所有处于 init 状态（或更高状态）邻居的路由器
   ID。要与邻居建立双向通信，路由器还必须接收邻居HELLO数据包的Neighbor字段中自己的路由器ID。有关详细示例和说明，请参阅为什么
   show ip ospf neighbor 命令显示邻居停滞在 Init 状态？

处于双向状态的邻居

   2-way状态表示路由器已在邻居HELLO数据包的Neighbor字段中收到自己的路由器ID。从处于init状态的邻居接收数据库描述符(DBD
   )数据包也会导致向2-way状态的转换。在广播和非广播多路访问(NBMA)网络中，OSPF邻居2-way状态不会引起担忧。有关 2-way
   状态的说明，请参阅为什么 show ip ospf neighbor 命令显示邻居停滞在 2-Way 状态？

处于 Exstart/Exchange 状态的邻居

   处于exstart或exchange状态的OSPF邻居尝试交换DBD数据包。路由器与其邻居形成主要和次要关系。邻接关系必须持续超过此状态。如果
   没有，则表示DBD交换发生问题，如最大传输单元(MTU)不匹配或接收到意外的DBD序列号。有关详细信息，请参阅为什么 OSPF 邻居停滞在
   Exstart/Exchange 状态？

处于 loading 状态的邻居

   在 loading
   状态下，路由器将发送链路状态请求数据包。在邻接时，如果路由器收到过时或丢失的链路状态通告(LSA)，它会发送链路状态请求数据包以请求LSA。未
   在此状态之后转换的邻居可能会交换损坏的LSA。此问题通常伴有 %OSPF-4-BADLSA 控制台消息。由于此问题不常见，请联系Cisco
   TAC寻求帮助。

   与美国邻居的相处之道

   美国人大多居住在独立的住宅中，邻里间的相处更突显美国人性格中独立的一面，当然这并不代表美国邻里间的关系是冷漠的，在很多时候美国邻居也是很热情的
   。

   如果你刚刚搬进一个住宅区，美国邻居会主动地到你家拜访，带一些他们自己制作的小点心来欢迎你，与你结识。即使这样，你也千万别把在中国对邻居那热情的
   一套方法原封不动地用在美国邻居身上，否则可能会引起不小的误会呢。


   1.见面打招呼

   中国人独特的打招呼的方法是“你吃了吗？”“去哪啊？”，见到邻居家小孩热情地问“多大了？”“长得多高了？”，再“热情”一点的连对方有几个孩子，孩
   子是否结婚，一个月挣多少都想问清楚了。但在美国千万别这样，会让美国人对你起疑心“他想干什么，不会有不好的想法吧”。这些问题在一些华人看来其实也
   就是顺嘴一问，并没有什么特别的意图，但这些都是对方家里的隐私，美国人是十分注重隐私权的，他们是不愿意回答你这些问题的。所以在美国见面打招呼微笑
   地说“hello”就好。

   2.去邻居家串门

   另外在中国，人们还喜欢好邻居家里串门，说说家长里短。但在美国不要在对方没有邀请你的时候就去串门。即便有事情要询问也不要突然造访，最好提前致电询
   问一下。并且在美国有一个称为“甜饼模式”的邻里想吃形式：如果有什么需要求助的或是提意见的事情，带上烤好的甜饼上门，借着送甜饼的机会委婉地说出自
   己的想法。
--
   3.送礼

   即使你已经和你的邻居足够熟了，你的美国邻居仍然会与你分得清楚，吃饭AA制，乘车AA制。这时你不要觉得难受，这只是美国人习惯的生活方式。你也不要
   为了处好关系就给你的美国邻居送礼，这反而会使得他们感到有负担。

   在一些情况下，特别是购买新开发的项目，邻居是谁根本无从考察，唯有在新居入伙后才能揭晓住在你隔壁的到底是恶邻居还是好邻居。

   一个恶邻居可能对你的生活造成巨大的困扰——平日的喧闹或凌晨聚会、不断吠叫的狗、来自隔壁猫咪的猫屎等琐碎但恼人的事情，都会打扰您和社区平静的日常
   生活。

   目前法律框架无法对付恶邻居

   依据目前的法律，邻居造成的麻烦可归为“滋扰”，属侵权行为。

   在所有沟通手段和方式，如与他们和平谈判、向相关部门投诉和向警方申报等，都无济于事的情况下，最后的办法就是进行法律诉讼。
--
   不过，并非人人愿意为这些琐碎事情向法庭提告，因为律师费昂贵且诉讼过程耗时。

   因此，我们认为，政府需要立法干预，解决恶邻居造成的社会问题。

   我们需要一套能以快速、低成本且有效的方式解决社区纠纷的法令。这这方面，我们可以参考新加坡《 2015年社区纠纷解决法》（简称新加坡CDRA）。

   “邻居”的定义

   若马来西亚要拟议相关新法令，应该参考新加坡CDRA，包含一些条例，例如，将该法令的适用范围局限在住宅产业上（新加坡CDRA适用于所有的“居住地
   ”）。

   此外，值得注意的是，新加坡CDRA中对“邻居”一词的诠释，是指与你居住在同一建筑物中，或者距离你的居住处100米内的住户，但不包括居住在同一单
   位内的人。

   干扰（滋扰）

   一般来说，定义滋扰侵权行为须先证明，是邻居的作为或不作为影响了邻居使用居住处的自由和乐趣。为了更好地说明，新加坡CDRA列出了10个邻居干扰（
   不详尽）的示例，即：

--
   2. 在居住处或附近乱丢垃圾

   3. 阻挡邻居的居住地

   4. 干扰邻居的可移动资产

   5. 在邻居住家附近，监视邻居和其住处

   6. 硬闯邻居居住地

   7. 允许动物侵入邻居居住地，并引起过多噪音或气味，或让小动物在邻居家周围排泄

   8. 以粗俗、带有侮辱或恐吓性的言语辱骂邻居

   9. 破坏公共设施和财产

   10. 进行造成邻居困扰和烦恼的恶作剧

   上述示例，可在准申诉人打算向法庭提告，并向造成滋扰的邻居索赔时，作为一定的指引。

   无论如何，法庭将根据每个案件的事实和情况，来决定某个行为是否属于滋扰或不合理的干预。
--
   我们认为，拟议新法律的核心是成立仲裁庭，这是解决邻里之间争端的有效平台。

   一个特别的仲裁庭可快速解决邻居的争端，因为与常规法院相比，其诉讼程序不那么正式，且较符合成本效益。与其他任何仲裁庭一样，除非仲裁庭有特别要求，
   否则无需聘请律师代表，诉讼方可自己提出辩论。

--
   是时候立法了

   就像拟议《住宅租赁法》一样，现在是时候让我们的立法者认真思考我们的提议，以解决恶邻居带来的社会问题。

   也许，对付恶邻居的法令可以和《住宅租赁法》合并，让其更加完善。但是，与任何法律制定过程一样，我们不应盲目采用其他国家的法律，忽略本地社会、文化
   和经济环境等的因素。

PyG教程(3)：邻居采样

   最新推荐文章于 2023-12-28 21:32:41 发布

   订阅专栏

一.为什么需要邻居采样？

   在GNN领域，大图是非常常见的，但由于GPU显存的限制，大图是无法放到GPU上进行训练的。为此，可以采用邻居采样，这样一来可以将GNN扩展到大
   图上。在PyG中，邻居采样的方式有很多种，具体详解torch_geometric.loader。本文以GraphSage中的邻居采样为例进行介
   绍，其在PyG中实现为NeighborLoader。

     NeighborSampler也是PyG中关于GraphSage中邻居采样的实现，但已经被弃用，在未来版本中会被删除。

二.NeighborLoader详解

2.1 GraphSage邻居采样原理

   假设采样的层数为 K K K，每层采样的邻居数为 S k S_k Sk​，GraphSage中邻居采样是这样进行的：
     * 步骤一：首先给定要采样邻居的小批量节点集 B \mathcal{B} B；
     * 步骤二：对 B \mathcal{B} B的 1 1 1跳（hop）邻居进行采样，然后得到 B 1 \mathcal{B}_1
       B1​，然后对 B 1 \mathcal{B}_1 B1​的 1 1 1跳邻居进行采样（即最初结点集的 2 2 2跳邻居）得到 B 2
       \mathcal{B}_2 B2​，如此往复进行 K K K次，得到最初小批量节点集相关的一个子图。

   下图左是GraphSage中给出的一个2层邻居采样的示例，其中每层采样的邻居数 S k S_k Sk​是相等的（图中为3）。

第二阶段：邻居发现协议与邻居缓存¶

IPv6 地址分类¶

   为了方便实验者阅读理解邻居发现协议（ND），本节首先对 IPv6 的地址分类进行一些介绍。

   IPv6 地址有三种类型，分别为单播地址、任播地址以及组播地址：
--
   在实践中，在网络层配置好 IP 地址后，网络节点还需要进行重复地址检测并加入相应组播组。在本实验中，这一过程可以省略。

邻居发现协议处理¶

   路由器工作在网络层（三层，采用 IP 协议），之下的数据链路层（二层，本实验基于以太网）为网络层提供 服务。网络层使用 IP
   地址标识不同的网络节点，而数据链路层则使用 MAC 地址（链路层地址）进行以太网帧的转发。因此，网络节点在发送和转发 IP 分组时，仅根据目的
   IP 地址确定下一跳 IP 地址是不够的。为了能够将 IP 分组通过以太网发送至下一跳节点，网络节点还需要知道下一跳的 MAC
   地址。邻居发现协议能够实现 IP 地址到 MAC 地址的转换这一地址解析（Address Resolution）过程。

   事实上，本实验中，实验者可以把 ND 协议处理实现在硬件中，也可以实现在软件中。本实验建议 ND
   协议采用硬件实现，其优点为方便调试，不需要等到 CPU
   及软件实现完毕后再进行调试；但是，如果软件需要访问邻居缓存（事实上，在本实验中一般不需要），则需要额外的工作。

   总体而言，实验者至少需要实现 ND 报文的以下处理，以便与其他网络节点互联互通：
--
       地址进行重复地址检测（Duplicate Address Detection，DAD），此时 ND
       报文的源地址为未指定地址。其中，实验者可以不实现对于后一种情况的处理，但是需要正确地丢弃这样的报文。
     * 单播 Neighbor Solicitation 接收与处理：其他节点正在针对该 IP 地址进行邻居不可达检测（Neighbor
       Unreachability Detection，NUD）。顺便指出，本实验中，实验者可以不实现单播 Neighbor
       Solicitation 的发送，即可以不主动进行 NUD，但是需要响应 NUD。
--
       Advertisement，由于其为组播，被当前节点接收。实验者需要正确地丢弃这样的报文。

   特别地，实验者可以不实现邻居缓存表项的状态维护（Neighbor Cache Entry
   States）。此外，实验者可以不实现对于任播地址，包括子网路由器任播地址的 Neighbor Solicitation 的接收与处理。

   邻居发现协议的更多细节请参见 RFC 4861: Neighbor Discovery for IP version 6
   (IPv6)，实验者可以重点关注第 4.3、4.4、4.6.1、7.1、7.2.2、7.2.3、7.2.4 以及 7.2.5 小节。

   Address），然后更新或插入到邻居缓存中。此处的更新是指，当邻居缓存中存在对应 IP 地址的条目时，更新其 MAC

邻居缓存设计与实现¶

   邻居缓存负责缓存 IP 地址与 MAC
   地址的对应关系。与转发表不同，邻居缓存的匹配算法采用精确匹配，因此较为简单。根据本实验的实际情况，邻居缓存所需支持的容量无需太大，十余条即可。
   实现邻居缓存时，实验者可以采用 CAM（Content-Addressable
   Memory）、蛮力查找或散列表等数据结构，同时可以加入过期自动删除的功能。其中，实验者也可以不实现过期自动删除的功能。

   实验者需要为邻居缓存设计好接口，写好 testbench，并进行充分测试。

如何应对邻居的骚扰？

   10月 22, 2020
   如何应对邻居的骚扰？ 如何应对邻居的骚扰？

   邻近的居住环境所面临的挑战之一，就是无法控制什么样的人住在你隔壁。幸运的话，你会有个爱好和平，彬彬有礼并尊重你隐私的邻居。

   但如果不幸遇到麻烦邻居呢？

   纽约和平研究所的项目主米歇尔.克士邦说：”因为人们在家的时间变多，听到噪音的机率变高，
   邻居之间对彼此越来越恼火。他们在家工作，孩子也在家，一切让人觉得困扰。”

   她指出，自从冠状病毒而需封城以来，纽约和平研究所的调解服务需求增加了。

   有些邻居制造的问题，我们可以忽略或容忍。像是垃圾发出异味或发出奇怪的声音，你可以要求房东根据租约来处理。

   但如果问题很严重，房东或楼管部门无法马上处理怎么办？又，如果你的疯狂邻居做出令人讨厌的行为，近乎于骚扰那该怎么办？

应对骚扰性邻居的4种方法

   如果邻居的行为已构成骚扰，那么就必须采取措施来解决–不仅仅是为了你的烦恼，也是为了你的安全。

1.   沟通

   如果你的邻居表现出不当行为，我们不建议你直接敲门对质。另外，在保持社交距离的大环境下，也不建议这样做。

   即使人与人之间的沟通因为疫情而更加困难，人们还是应该采取不需要调解的情况下解决问题。

   你可以给你的邻居写一封电子邮件或手写信。前提是你必须知道对方愿意合作，只需要和他谈谈就能解决问题。你并不需要成为无理取闹的一方，理想是不费吹灰
   之力就能解决问题。

   如果最初联系邻居的方式没有成功，那么可以把问题交由第三方处理。

2.   与业主或董事会协商
--
   理想情况下，房东和合作委员会应该尽其所能，使你的居住条件安全舒适。他们可以对孩子们在走廊上跑来跑去、抽烟、晚间大声聚会等问题进行调解。

   如果是垃圾发臭造成害虫的问题，那么房东一定要介入。但有时情况会变得很棘手–有的房东和合作委员会宁可不介入邻居内部的问题，因为他们无力驱逐个人或
   使其住户离开。

--
   这是寻求法律诉讼前的一个步骤。使用调解的方式其实更快，费用更低，在某些州调解甚至是免费的。这是一个对抗性较低的方式，只要双方选择参与解决问题。

   与其把事情闹上法庭，你和邻居可以想出一些双方都能接受的办法。

   现在由于疫情的影响，让双方同意调解是很困难的。就警局而言，由于封城，他们的服务已经远程化但他们也设计了变通方法，利用视频会议进行调解。
--
   是否构成骚扰条件取决于各州的法律，并重于当事者企图。如果情况失去控制，就是让警察介入的时候。有时只需要一个警察，就可以让当事者就范。

   如果需要提出刑事申诉，让邻居被拘捕，可以去当地警察局要求法院下达命令，使邻居远离你和你的财产。

   如果涉及的骚扰实际上是对您安全的威胁，则可以考虑采取另一种方式，例如向安全小组寻求帮助以评估威胁。

   住在一处，他们打破了关起门各自过日子的邻里状态，各家大门常常敞开着，把商品房住出了大院的感觉。

--

   此生活中重要的“家人”。

   “快节奏的生活中大家都很忙，就算是邻居，可能也不会有太多交际。很多人在一个地方住了很多年，都不知道自己的邻居是谁。”住在1608的熊伟回忆，邻
   里间的相熟开始于彼此的孩子。由于4个家庭的孩子都在同一小学上学，加之年龄相仿，孩子们很快便打成了一片。

--
   成为家人，没有血缘关系却胜似亲人

   为了方便交流，4户邻居建起微信群，家住1605的彭华年龄稍长，总会在每个节假日做好具有家乡特色的菜肴，在群内招呼邻居们一起来家中过节。每逢节日
   ，4家人就像一个大家庭般相聚在一起，共同庆贺。“有时候我们还会一家做一两个菜，大家凑在一起吃。”为了方便相互沟通交流，4家人的大门总是敞开着。

--

   张家界、武汉、成都、西安……全国各地都留下了这个大家庭的笑声与回忆。“可惜的是，住在1606的马杜一家，因为工作原因搬到四川去了。”说起邻居的
   离开，熊伟感觉有些可惜，所幸对方一家人依旧在微信群内活跃，还相约过年时相聚。“新搬来的邻居奶奶人也特别好，我们相处得很开心。”熊伟告诉三湘都市
   报记者，4户人家的相处，让她重新定义了邻里情。

   “这是我们认识的第8年，是日月交替的一个个春夏秋冬!”熊伟在朋友圈发出感慨，配图是4户邻居8年来相处的点点滴滴，在她的心中，这群邻居虽然没有血
   缘关系，却胜似亲人。

   按照这些提示成为一个更好的邻居

   多年来社区发生了变化。 过去，人们在街区上为新人们带来了三明治或水果篮。 有时候他们会提出帮助一个项目。

   即使在今天，所有的街区都有自己的个性。

   有些人倾向于对街头派对和社交更加开放，而另一些则倾向于留在自己身边的人组成。 第二组不坏， 它只是不那么热情和邀请。
   他们中的许多人要求苛刻的工作和其他责任，所以他们没有时间与邻居进行社交。

   不管你居住的邻居类型如何，你仍然应该遵循一些基本的礼节规则来成为一个好邻居。
   最终目标是生活在和平与和谐之中，尽管这并非总是可行，但至少可以采取一切措施来防止敌对行为和长期坚持的感情。

--

   没有人希望你像老鼠一样安静，但当谈到噪音太大时，你也应该尊重某些基本规则。
   很大程度上取决于你住在哪里，你住的是什么类型的住宅，以及你邻居的习惯。 在充满年轻家庭的街道上的噪音可接受性与单身二十几岁的公寓大楼不同。

   了解一些噪音礼节规则：
     * 如果你住在一栋公寓楼，不要对大厅另一端的人说话。
     * 没有理由在晚上8点以后与隔壁公寓共用的普通墙上钉钉子。 它可以等到第二天。
     * 星期六早上7点割草坪不会让你爱上你的邻居。 等到至少10点。

     * 每次你进入车道时都不要鸣喇叭。 你的家人很快就会发现你很快就回家了。
     * 和你的配偶或朋友在温暖的夏日夜晚外面散步真是太棒了，但是如果你的房子很小，而邻居的卧室窗户很近，那么在晚上10点后面的门廊里进行长时间
       谈话是无礼的。所以你的邻居可以睡觉。
     * 从晚上9点到早上9点避免任何不必要的噪音。这包括加速汽车引擎，骑摩托车的儿童，大声说话以及其他任何可能唤醒某人睡眠的东西。

做一个好邻居

   遵循黄金法则成为一个好邻居。 这些是你可能每天都会看到的人，或者至少是经常看到的人。 在你知道自己行为不当之后，你不想羞辱自己。

   作为一个好邻居的提示包括：
     * 观察并尊重你邻居的个人空间 。
     * 尽量不要借用任何东西 ，但如果必须，请在使用后立即退还该物品。 如果你打破这个项目，付出修理或更换它。
     * 不要成为邻居的八卦 。 这只是粗鲁，最终会回来咬你。
     * 如果你与邻居有问题，直接转到该人并以成人的方式讨论。 除非你受到威胁，否则不要打电话给警察。

     * 不是每个人都是狗或猫爱好者，因此对你的宠物负责。 这包括让他们离开邻居的草坪并追随他们。
     * 在假期期间用卡片或小型自制礼物记住你的邻居，不要期望任何回报。 享受给予的行为 。
     * 如果你和邻居有误解，可以通过握手和至少友好条款来做出正确的决定。 你不必闲逛。 当你进入车道时，一个简单的波就足够了。
     * 如果您的社区拥有房主协会，请了解规则并遵守它们。

遏制上诉

   一个好邻居是维护房子和草坪外部的人，至少与邻里的其他地方保持同一水平。 你不希望你的房子成为街道上拖拽房屋价值的房子。

   如果你不能修剪自己的草坪，可以租一个邻居的小孩或草坪服务来保持它的整洁。

   遵循一些基本的外部维护：
     * 修剪草坪。 在做一些事情之前不要让它变得很高。
     * 必要时从外墙上取下模具并涂漆。
     * 确保树木，灌木和其他元素不会蔓延到邻居的院子里。 这包括树枝，蔓延灌木和杂草。
     * 在一天结束时，从前院取出所有自行车，滑板和玩具。

社会事件

   你可能是派对动物，只要你不闯入或强加给你的邻居，这就好了。 偶尔的深夜爆炸可能会被原谅，但每个周末的通宵派对都会让你成为坏人。
   派对时要尊重邻居。

   了解一些好邻居社交指南：
     * 请您的客人停在车道上或房子前面。 确保他们不会阻挡别人的车道，或在公寓大楼中使用其他居民的指定地点。
     * 服从噪音法令。 如果邻居打来电话或要求你冷静下来，请友好并道歉 。 然后安静下来。
     * 邀请邻居参加你的大派对。 这样做显示了你的友善，而且他们会比你没有被邀请并且不得不听你客人的乐趣声音好得多。

欢迎新邻居

   邻居们可以通过友好的介绍让新家庭感受到欢迎。 这丰富了新邻居的生活以及在那里居住了一段时间的人们的生活。

   欢迎新邻居的想法包括：
     * 尽快介绍自己 。 给新邻居你的电话号码。
     * 提供膳食和治疗。
     * 从您的花园递送一束鲜切花 。
     * 当搬家搬运家具时，提议让新邻居的孩子过来。
     * 为新家庭提供该地区的服务和电话号码列表。 包括紧急服务，医生办公室，牙医，学校，提供结转的地方，干洗店，以及任何你能想到的东西。

     * 每年举行一次街区派对来了解所有新邻居。

   当人们通过适当的礼节努力成为好邻居时，每个人都会有一种幸福感，知道你们都被朋友包围着 。 这个社区会更有吸引力，你的家会更加满足你的需求。
   一个额外的好处是知道你和你的邻居有对方的背部。

邻居发现

   May 11, 2023

   C

   邻居发现 (ND) 是 IPv6 最重要的协议之一。它是一种基于消息的协议，结合了地址解析协议 (ARP)、互联网控制消息协议 (ICMP)
   和路由器发现的功能。ND 允许节点通告其链路层地址并获取相邻节点的 MAC 地址或链路层地址。此过程由邻居发现协议 (ND6) 执行。

   邻居搜索可以执行以下功能：
     * 路由器发现：使主机能够在连接的链路上发现本地路由器并自动配置默认路由器。
     * 前缀发现：使主机能够发现本地目标的网络前缀。
--
       IPv6 地址的地址自动配置。
     * 地址解析：等同于 IPv4 中的 ARP，允许节点将相邻节点的 IPv6 地址解析为其链路层地址。
     * 邻居不可访问性检测：使节点能够确定邻居的可访问性状态。
     * 重复地址检测：使节点能够确定相邻节点是否已在使用 NSIP 地址。
     * 重定向：等同于 IPv4 ICMP 重定向消息，允许路由器将主机重定向到更好的第一跳 IPv6 地址以到达目的地。
